# Contributing to Iskra

Дякуємо за бажання долучитись! Цей проєкт навмисно простий — чистий HTML/CSS/JS,
без збірників і фреймворків — щоб будь-хто міг відкрити код і одразу зрозуміти, що відбувається.

## Локальний запуск

```bash
git clone https://github.com/<your-username>/iskra.git
cd iskra
npm install
npm start
```

Сайт підніметься на `http://localhost:5173`.

Можна й без npm — просто відкрити `src/index.html` у браузері.

## Структура проєкту

```
iskra/
├── src/
│   ├── index.html
│   ├── css/style.css
│   └── js/
│       ├── api.js   ← мок-шар "бекенду" (див. docs/DATABASE.md)
│       └── app.js   ← логіка інтерфейсу
├── docs/
│   └── DATABASE.md  ← як підключити реальну базу даних
└── .github/workflows/
```

## Правила розробки

- **UI-код (`app.js`) не має напряму торкатись даних** — тільки через функції `api.js`.
  Це дозволяє замінити мок-бекенд на реальний, не чіпаючи інтерфейс.
- Тримайся дизайн-токенів у `:root` в `style.css` (кольори, шрифти, радіуси) — не хардкодь нові значення.
- Комітьте invididual, логічно завершені зміни; описовий commit message (укр. або англ. — обидва ок).
- Перед PR прогони `npm run lint:html`.

## Як запропонувати зміну

1. Форкни репозиторій і створи гілку: `git checkout -b feature/назва-фічі`.
2. Внеси зміни, переконайся, що сайт і далі відкривається без консольних помилок.
3. Відкрий Pull Request з коротким описом: що і навіщо змінено.
4. Якщо зміна впливає на UX (нові екрани, форми) — додай скріншот у опис PR.

## Повідомлення про баг

Використовуй шаблон issue "Bug report" — вкажи кроки відтворення, очікуваний і фактичний результат, браузер/ОС.

## Кодекс поведінки

Будь доброзичливим і конструктивним. Ми тут, щоб робити хороший продукт разом.
